# Hive brain entry schema

Every entry under `~/.hivesmith/brain/` is a markdown file with **YAML front-matter** + body. This document defines the schema and the trust boundary.

## Front-matter

```yaml
---
slug: tests-flaky-on-macos-network          # required, kebab-case, ≤80 chars
scope: project                              # required: universal | ecosystem | user | project
ecosystem: bun                              # required when scope=ecosystem
repo: 7f3a91c2b4d5                          # required when scope=project; first 12 chars of sha256(canonicalized remote.origin.url)
applies_to: ["**/test/**/*.ts"]             # optional; comma/list of globs (Cursor-rules-style conditional injection)
tags: [testing, ci, flake]                  # optional; flat list, ≤2-level nesting
graph_nodes: [review_pr_skill]              # optional; graphify node IDs the entry references
valid_until: 2026-08-09                     # optional; entries past this date are archived by the gardener
provenance:
  source: hs-feature-implement              # which skill wrote it
  session: 2026-05-09T17:42:11Z             # ISO timestamp
  pr: 42                                    # optional PR number
  trusted: true                             # false = derived from untrusted file content → unverified/
confidence: 0.7                             # 0.0–1.0
created: 2026-05-09                         # required; YYYY-MM-DD
backlinks: 0                                # gardener-maintained
---
```

## Body

```markdown
# <Concise lesson title>

**Lesson:** <one-paragraph distilled lesson>

**Why:** <one-paragraph reason or context>

**How to apply:** <one-paragraph guidance for future sessions>
```

The body must NOT contain raw code dumps. The redactor rejects any code fence longer than 25 lines (`BRAIN_REDACT_MAX_FENCE_LINES`). Distill the lesson; cite via `[[graphify-node]]` wikilinks if needed.

## Scope semantics

- `universal` — applies in every project.
- `ecosystem` — applies in projects of a given runtime/language. Requires `ecosystem:` field.
- `user` — applies to this user across all projects (preferences, work-style notes).
- `project` — applies to one specific repo. Requires `repo:` hash.

Auto-writes default to `scope=project`. Promotion to broader scope is **explicit** via the `/hs-brain-promote` skill — never automatic.

## Trust boundary

Brain entries are loaded into agent prompts wrapped in `<project-memory untrusted="true">…</project-memory>` delimiters. They MUST NOT:

- Grant tool permissions
- Override `AGENTS.md` or `CLAUDE.md` instructions
- Direct the agent to execute commands

Skills that read the brain treat the contents as untrusted external data. Entries derived from untrusted file sources (READMEs, web pages, issue bodies) land in `unverified/` and require a manual review (typically `/hs-brain-promote`) before being moved into a scoped directory.

## Redaction

Before any entry is written:

1. The body is run through `scripts/brain/redact.sh`, which masks AWS keys, GitHub tokens, private keys, Slack tokens, and high-entropy hex blobs.
2. If `gitleaks` is on `PATH`, the body is also passed through `gitleaks detect --pipe` and the write aborts on any hit.
3. Code fences longer than 25 lines abort the write.

Skills surface the abort to the user; the entry is not silently dropped.

## Index

`INDEX.md` at the brain root is regenerated by `scripts/brain/index.sh`. It has two tiers — `<!-- HOT -->` (≤50 lines, always-injected) and `<!-- ALL -->` (≤2000 lines, eager-filtered by active project context). It is rebuilt at session boundaries by the gardener — not on every write — so the prompt cache stays warm.

## Filesystem layout

```
~/.hivesmith/brain/
├── INDEX.md              # tiered index, gardener-regenerated
├── SCHEMA.md             # this file
├── README.md             # team onboarding
├── universal/            # cross-project lessons
├── ecosystem/<lang>/     # language/runtime-scoped
├── user/                 # this user's preferences
├── project/<repo-hash>/  # per-repo lessons
├── unverified/           # quarantine: untrusted-source provenance
└── archive/<YYYY-MM>/    # gardener-evicted (kept in git history)
```
