- **`/review-pr` fans out on risk surface instead of diff size, and the orchestrator's own read now does real work.** The orchestrator used to read the diff, `$META`, `$THREADS`, `AGENTS.md` and the hive brain, then discard that reading and pay 2–4 subagents to re-derive it, with the fan-out shape keyed off a LOC tier table — a poor proxy for what a review needs, since a 40-line auth change needs a security specialist and a 900-line generated bump needs almost nothing. Now a new §1.5 baseline pass reviews the diff in the context already loaded, against all four dimension checklists, establishing a coverage floor on every PR at zero extra input tokens. §2 replaces the tier table with a surface-trigger gate: each dimension dispatches only when the diff contains its surface, ambiguity resolves toward dispatch, and skipped dimensions are named with a reason in a `DISPATCH:` line rather than silently dropped. Dispatched agents now do two things rather than one: their full dimension checklist against the diff (they are the dedicated specialist and the second independent look — the baseline is one reader multiplexing four dimensions, so it is a floor, not a ceiling) and then investigation **beyond** the diff — grepping call sites, opening implementations, reading files the diff never touches — which is work the baseline structurally cannot do. The Security trigger fires on any dependency or pinned-action bump even when only `config`/`ci` files change, so a lockfile or `uses: foo@v4 → @v6` PR still gets a supply-chain review rather than falling through the gate as "only config". §5.2 previously dropped any finding whose `file:line` fell outside the diff range; that rule now resolves citations against the working tree, because a broken caller in an unchanged file is the highest-value finding in the review and was being deleted before it reached the output. Baseline findings are exempt from the §5.3 spot-verify re-read (already read in context) and flow into the same dedup. Tier vocabulary is purged from §6 output and the §7 rubric; a dimension skipped by the gate no longer degrades the verdict, while a dispatched agent that fails still does. A new `cross-file` fixture case covers the regression the existing three could not see — every planted bug in `planted-bugs` sits in a single file, so nothing in the suite exercised out-of-diff breakage. Model pinning from #42 (`hs-reviewer` on sonnet, Security overridden to opus) is unchanged. Net cost: thin and mid-size PRs get cheaper (docs-only drops to zero agents, a single-surface change to one); a PR spanning all four surfaces costs more and is reviewed more thoroughly.
