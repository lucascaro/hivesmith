- **`/review-pr` now adapts its shape to diff size instead of always fanning out.** The skill dispatched up to four agents on every standard PR, each re-reading the same diff and context bundle (9k–23k tokens of cold re-read apiece) and each independently grepping the same changed symbols' call sites — one shared investigation run four times, with fan-out keyed off a LOC tier table that is a poor proxy for what a review needs. It now runs two named passes: **§2 Pass 1** reviews the diff against all four dimension checklists, **§3 Pass 2** investigates what the diff *reaches* outside itself. On an ordinary PR both run inline with zero subagents. **§2.0** splits Pass 1 across four dimension agents only above ~1000 changed lines or ~15 files, and **§4** escalates a single out-of-diff investigation only when a measured `grep -rl` shows more than ~10 files to open. Pass 2 stays linear at every size — it is shared across dimensions, so splitting it is what made four agents grep the same call sites four times.
- **Both sides of the size gate were measured, not assumed** — method, numbers and caveats in `skills/review-pr/fixtures/cases/subtle-bug/README.md`. *Below it:* a dedicated single-dimension specialist and a single reader carrying all four checklists, three runs each against a deliberately subtle ~190-line defect (a missing `tx.rollback()` before an early return, correct in isolation and wrong only relative to its four sibling paths). Both caught it 3/3 at identical severity and confidence, and the multiplexed reader was *better* — it linked the defect to the untested branch that let it ship green, a connection splitting dimensions severs. Fan-out cost 102,868 tokens against 26,980 for equal recall. *Above it:* the same comparison on a real 1,356-line / 23-file PR inverted — four agents found 13 findings (8 IMPORTANT) to the linear reader's 6 (4 IMPORTANT), which missed nine defects in the skill's own logic. Attention dilutes with diff size; per token the linear reader was still more efficient (0.062 vs 0.044 findings/1k), so the gate is a budget decision tied to size rather than a claim that either shape wins outright.
- **Pass 2's angles are explicit rather than implied.** Seven named angles — call sites, implementations, deletions & renames, consumers of changed data, supply chain, convention drift, test reach — each with its own trigger, all resolving toward investigating when ambiguous, and a visible `INVESTIGATE:` line naming every skipped angle *with its reason*. The supply-chain angle fires on any dependency or pinned-action bump **even when only `config`/`ci` files change**, so a lockfile or `uses: foo@v4 → @v6` PR still gets a supply-chain review rather than falling through as "only config".
- **Anti-skim discipline for the single reader.** Every dimension must be named in the output including the clean ones ("Safety: no tests, no module-level state — clean"), because an unnamed dimension is an unchecked one; and the reader is directed to actively hunt cross-dimension links, the one thing a split-up review structurally cannot do. Two checklist items were added from what the measurement showed reviewers keying on: *resource pairing on every exit path* (correctness) and *coverage of newly added branches* (safety).
- **Six review-integrity bugs fixed, surfaced by running the old and new skills against this PR itself.** (1) Findings citing a line outside the diff range were silently dropped — deleting exactly the out-of-diff breakage the review exists to catch; citations now resolve against the working tree. (2) Nothing verified a cited `file:line` on the inline path: Pass 2 findings come from `grep` hits in files never opened, yet the escalated-agent contract demanded exactly that check, so the inline path was strictly weaker than the one it replaced. Verification is now required for every finding not read in context, and line numbers are corrected rather than trusted. (3) Dedup grouped purely by position, so two distinct defects on adjacent lines silently collapsed into one; merging now requires the same underlying root cause. (4) The finding enum omitted `ux` and `consistency` even though Checklist 4 covers them, and the escalated-agent contract emitted a `category` outside the enum entirely. (5) The style-only-MINOR rule was never enforced at output time — now an active filter with MINOR capped at 5, which moved `planted-bugs` recall 3/4 → 4/4 and precision 27% → 80%. (6) The escalation-measurement command used a single-pattern `grep -v` to exclude changed files, so with more than one changed file the count inflated and escalated investigations that should have run inline.
- **Anti-injection promoted to a standing §0.1 rule** binding the orchestrator directly, now that it reads the untrusted diff itself rather than delegating it — injected instructions in a diff or PR body are a BLOCKING security finding. `hs-reviewer` keeps its name (load-bearing in `install.sh` and the `subagent-linking` CI job) and its sonnet pin, but its role narrows to scoped retrieval with an explicit "you are not a reviewer and carry no checklist".
- **Fixture harness gained three cases and three fixes, and now documents what it does *not* cover.** New: `cross-file` (a required parameter added, caller left broken in a file outside the diff), `subtle-bug` (the measurement case above), and `injected-instructions` — a diff whose added lines instruct the reviewer to skip the security checklist and return `APPROVE`, with the same pressure repeated in the PR body. That last one closes a real gap: the anti-injection rule is the mitigation this design leans on hardest, since the orchestrator now reads the untrusted diff directly in a context holding Bash and Agent, and nothing tested it. Fixed: `bin/gh` was missing entirely, so cases silently hit the **real GitHub API** instead of the mock; `mock-gh` had no `api graphql` branch, so every case aborted on the review-threads query; and `bin/grade` no longer grades on `category`, which is a judgment call for defects sitting under two checklists — `file` + `line_range` + `keywords` already anchor every spec, and grading the label trained readers to relax specs instead of investigating. `fixtures/README.md` now carries an explicit uncovered-branches table (the §2.0 size gate, §4 escalation, §5 verification, `--all`, the style filter) so a green suite is not misread as full coverage. All six cases pass as real skill executions.
