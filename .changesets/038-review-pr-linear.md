- **`/review-pr` is now a single linear pass instead of a per-dimension fan-out.** The skill dispatched up to four agents, each re-reading the same diff and context bundle (9k–23k tokens of cold re-read apiece) and each independently grepping the same changed symbols' call sites — one shared investigation run four times, with fan-out shape keyed off a LOC tier table that is a poor proxy for what a review needs. It now runs two named passes in one context: **§2 Pass 1** reviews the diff against all four dimension checklists, **§3 Pass 2** investigates what the diff *reaches* outside itself. Subagents are gone from the default path; **§4** escalates a single oversized investigation to `hs-reviewer` only when a measured `grep -rl` shows more than ~10 files to open, and that agent now receives one concrete retrieval task rather than a review dimension.
- **The collapse was measured, not assumed** — method and numbers in `skills/review-pr/fixtures/cases/subtle-bug/README.md`. A dedicated single-dimension specialist and a single reader carrying all four checklists were run three times each against a deliberately subtle defect: a missing `tx.rollback()` before an early return, buried in ~190 lines of benign churn, correct in isolation and wrong only relative to the four sibling paths that do roll back. Both arms caught it 3/3 at identical severity and confidence, and the all-dimensions reader was *better* — it linked the defect to the untested branch that let it ship green, a correctness×test-hygiene connection the specialist arm mostly missed because splitting dimensions severs links that cross them. Fan-out cost 102,868 tokens against 26,980 for equal recall.
- **Pass 2's angles are explicit rather than implied.** Seven named angles — call sites, implementations, deletions & renames, consumers of changed data, supply chain, convention drift, test reach — each with its own trigger, all resolving toward investigating when ambiguous, and a visible `INVESTIGATE:` line naming every skipped angle *with its reason*. The supply-chain angle fires on any dependency or pinned-action bump **even when only `config`/`ci` files change**, so a lockfile or `uses: foo@v4 → @v6` PR still gets a supply-chain review rather than falling through as "only config".
- **Anti-skim discipline for the single reader.** Every dimension must be named in the output including the clean ones ("Safety: no tests, no module-level state — clean"), because an unnamed dimension is an unchecked one; and the reader is directed to actively hunt cross-dimension links, the one thing a split-up review structurally cannot do. Two checklist items were added from what the measurement showed reviewers keying on: *resource pairing on every exit path* (correctness) and *coverage of newly added branches* (safety).
- **Three real bugs fixed, all surfaced by running the fixture suite.** Findings citing a line outside the diff range were silently dropped — deleting exactly the out-of-diff breakage the review exists to catch, such as a caller left broken in an unchanged file; citations now resolve against the working tree. Dedup keyed on `(file, line, category)`, so one defect legitimately labeled under two checklists — a data race is both a concurrency bug and a shared-mutable-state bug — split into two findings; it now keys on `(file, line)` and `category` is documented as informational. And the "don't flag style-only issues" rule was never enforced at output time, so a single reader running four checklists reported everything it saw; it is now an active filter with MINOR capped at 5. On the `planted-bugs` fixture the last fix moved recall 3/4 → 4/4 and precision 27% → 80%.
- **Anti-injection promoted to a standing §0.1 rule** binding the orchestrator directly, now that it reads the untrusted diff itself rather than delegating it — injected instructions in a diff or PR body are a BLOCKING security finding. `hs-reviewer` keeps its name (load-bearing in `install.sh` and the `subagent-linking` CI job) and its sonnet pin, but its role narrows to scoped retrieval with an explicit "you are not a reviewer and carry no checklist".
- **Fixture harness gained two cases and three fixes.** New `cross-file` (a required parameter added, caller left broken in a file outside the diff) and `subtle-bug` (the measurement case above) cover axes the existing three could not — every planted bug in `planted-bugs` sits in a single file. Fixed: `bin/gh` was missing entirely, so cases silently hit the **real GitHub API** instead of the mock; `mock-gh` had no `api graphql` branch, so every case aborted on the review-threads query; and `bin/grade` now accepts `|`-alternation for defects that legitimately sit under more than one checklist. All five cases pass as real skill executions.
