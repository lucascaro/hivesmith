- **Decentralize merge-conflict-prone aggregate files.** `CHANGELOG.md`'s `[Unreleased]` body, `docs/product-specs/index.md`, and `docs/exec-plans/tech-debt-tracker.md` are now **generated** from per-PR sources by `scripts/regen-generated.sh`. Each PR writes one `.changesets/<NNN>-<slug>.md` plus updates to the relevant spec's YAML frontmatter (`stage:`, `pr:`, `shipped:`); a GitHub Action regenerates the aggregates on push to `main` and direct-pushes the result with `[skip ci]`. A second Action (`block-generated-edits`) hard-fails PRs that touch the generated files directly; bypass with the `regen-override` label. A third Action (`verify-generated`) runs the regenerator against the merge ref to catch interaction bugs before they land. Spec frontmatter is the sole source of truth for `stage:` â€” the exec-plan `Stage:` line is removed from the template and from every existing plan. Skill writes are reordered so the stage transition is always the **last** write of a multi-write sequence, making partial-state crashes resumable by the next skill's cold-start guard. Sort order is filename-only (`NNN-` prefix), monotonic and append-only, so adding a changeset never rewrites lines outside its own type section. Migration tools (`scripts/migrate-to-changesets.sh`, `hivesmith doctor`) ship for downstream projects. 13 skills updated: `changelog-update`, `feature-{new,triage,research,plan,implement,qa,populate-backlog,loop,next}`, `release`, `gc-sweep`, `hivesmith-init`.

  **Migration for downstream projects.** To migrate an existing hivesmith-using project to the decentralized layout, tell your agent:

  > Run `scripts/migrate-to-changesets.sh` at the repo root (it splits the current `CHANGELOG.md` `[Unreleased]` body into `.changesets/<NNN>-<slug>.md` files). Add YAML frontmatter to every file under `docs/product-specs/` (the existing index table is the data source: `issue`, `title`, `type`, `complexity`, `priority`, `stage`, plus `pr` + `shipped` for completed items). Verify `scripts/regen-generated.sh --check` exits 0. Open a PR titled "chore: migrate to decentralized indices" with the `regen-override` label so the new `block-generated-edits` job lets the regenerated `CHANGELOG.md` / `docs/product-specs/index.md` / `docs/exec-plans/tech-debt-tracker.md` land. Then delete any hand-written entries in those generated files â€” they will be rebuilt on the next push to `main`.

  For contributors with in-flight branches: rebase onto `main` after this lands, drop any conflicts in `CHANGELOG.md` / `docs/product-specs/index.md` (those edits are now obsolete), and add a `.changesets/*.md` describing your change. Use the `no-changeset` PR label if the change is docs- or CI-only.
