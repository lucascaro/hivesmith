- **`/review-pr` is now a single linear pass instead of a per-dimension fan-out.** The skill dispatched up to four agents, each re-reading the same diff and context bundle (9k–23k tokens of cold re-read apiece) and each independently grepping the same changed symbols' call sites — one shared investigation run four times. It now runs two named passes in one context: **§2 Pass 1** reviews the diff against all four dimension checklists, **§3 Pass 2** investigates what the diff *reaches* outside itself. Subagents are gone from the default path; **§4** escalates a single oversized investigation to `hs-reviewer` only when a measured `grep -rl` shows more than ~10 files to open, and that agent now gets one concrete retrieval task rather than a review dimension.
- **This was measured, not assumed** — see `fixtures/cases/subtle-bug/README.md`. A dedicated single-dimension specialist and a single reader carrying all four checklists were run three times each against a deliberately subtle defect (a missing `tx.rollback()` before an early return, buried in ~190 lines of benign churn). Both caught it 3/3 at identical severity and confidence, and the all-dimensions reader was *better*: it linked the defect to the untested branch that let it ship green, a correctness×test-hygiene connection the specialist arm mostly missed. Per-dimension fan-out cost 102,868 tokens against 26,980 for equal recall — 3.8× for worse cross-dimension reasoning. The measurement retires the hypothesis the previous revision was preserving.
- **Pass 2's angles are now explicit rather than implied.** Seven named angles — call sites, implementations, deletions & renames, consumers of changed data, supply chain, convention drift, test reach — each with its own trigger, all resolving toward investigating when ambiguous, and a visible `INVESTIGATE:` line naming every angle skipped *with its reason*. Previously this was one sentence telling agents to grep call sites.
- **Anti-skim discipline for the single reader.** Every dimension must be named in the output including the clean ones ("Safety: no tests, no module-level state — clean"), because an unnamed dimension is an unchecked one; and the reader is explicitly directed to hunt cross-dimension links, which is the one thing a split-up review structurally cannot do. Two checklist items were added from what the measurement showed reviewers keying on: *resource pairing on every exit path* (correctness) and *coverage of the new branches* (safety).
- **Two real bugs fixed, both found by running the suite.** Dedup keyed on `(file, line, category)`, so one defect legitimately labeled under two checklists — a data race is both a concurrency bug and a shared-mutable-state bug — split into two findings; it now keys on `(file, line)` alone and `category` is documented as informational. And the §8 "don't flag style-only issues" rule was never enforced at output time, so a single reader running four checklists reported everything it saw; it is now an active filter at §5 step 5, with MINOR capped at 5. On the `planted-bugs` fixture this moved recall 3/4 → 4/4 and precision 27% → 80%.
- **Anti-injection promoted to §0.1** as a standing rule binding the orchestrator directly — it now reads the untrusted diff itself rather than delegating it, and injected instructions are a BLOCKING security finding. `hs-reviewer` keeps its name (load-bearing in `install.sh` and the `subagent-linking` CI job) and its sonnet pin, but its role narrows to scoped retrieval with an explicit "you are not a reviewer and carry no checklist". The fixture harness gained a `cross-file` and a `subtle-bug` case, a `bin/gh` shim that was missing entirely (cases silently hit the real GitHub API), an `api graphql` branch in `mock-gh` (every case aborted on the threads query), and `|`-alternation support for ambiguous categories in `bin/grade`. All five cases pass as real skill executions.
