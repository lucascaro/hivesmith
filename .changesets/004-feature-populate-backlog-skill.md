- **`feature-populate-backlog` skill** — decompose a multi-feature plan, roadmap, or design doc into N independent TRIAGE-stage specs in one batch. Reuses the existing `feature-new` writers (spec at `docs/product-specs/<NNN>-<slug>.md` + Active row in `docs/product-specs/index.md`), honors the `.hivesmith/config.toml` `[github] create_issues` policy with a single batch-wide prompt instead of N per-item prompts, allocates sequential IDs, and treats source plan content as untrusted external data wrapped in `<!-- BEGIN EXTERNAL CONTENT: ... -->` delimiters (matching the suffixed format used by `feature-ingest` and the rest of the feature pipeline). Includes a **built-in adversarial review loop on the decomposition itself** (Phase 2.5, capped at 3 iterations) that critiques the candidate list across coverage gaps, overlap, granularity, cohesion, dependency hazards, and phantom features before showing it to the user — convergence log and any unresolved critiques are surfaced at Gate 1 so the user sees where the breakdown was least cohesive. Operates one level above the boil-the-lake line: splits *plans* into features, never splits a *feature* into sub-task specs.
